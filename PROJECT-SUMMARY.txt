================================================================================
WHAT THIS PROJECT IS
================================================================================

A beautiful, retro-styled voting system where users can:
- Submit theme suggestions (max 50 characters)
- Vote on themes with upvotes
- View all themes sorted by popularity
- See completed themes marked as "DONE"

Admin panel allows you to:
- Mark themes as complete (with checkbox)
- Edit theme text
- Delete themes
- View statistics

Design: Black & white retro aesthetic with Times New Roman font

================================================================================
DATABASE OPTIONS (AUTO-DETECTED)
================================================================================

Your code automatically chooses in this order:

1. EDGE CONFIG (if EDGE_CONFIG env var exists)
   - Ultra-fast reads (< 1ms)
   - Slower writes (1-2 second delay)
   - Best for: High read, low write scenarios

2. POSTGRES (if POSTGRES_URL env var exists)
   - Balanced read/write performance
   - Proper SQL database
   - Best for: Most use cases

3. KV (if KV_REST_API_URL env var exists)
   - Fast Redis-based storage
   - Good middle ground

4. TURSO (if TURSO_DATABASE_URL env var exists)
   - External SQLite for edge

5. LOCAL SQLITE (default for development)
   - No setup needed locally

Just set the environment variables for your preferred option!
